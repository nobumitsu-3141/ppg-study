# decklib.py の 4 分岐について

`decklib.py`（川副式スライド ビルダー）は共有ライブラリとして設計されているが、
**実際にはデックごとにコピーされ、独立に育っている**。移行時点の実測は次のとおり。

| デック | 関数数 | サイズ | 更新日 | 位置づけ |
|---|---|---|---|---|
| `defib` | 28 | 24,350 B | 2026-07-17 | **最新・最大。正典として `lib/decklib.py` に採用** |
| `aline` | 27 | 23,564 B | 2026-07-11 | defib に次ぐ |
| `etco2` | 25 | 20,382 B | 2026-07-11 | |
| `monitor-ecg` | 16 | 10,993 B | 2026-07-11 | 最小。bullets / column / menu / divider を持たない |

## 関数名の包含関係

`defib` 版は他 3 版の関数名をすべて含む**上位集合**である。`defib` だけが持つのは
`_title_outline`（タイトルに黒 2.25 pt の縁取りを付与。コメントに「v7（2026-07-15）」とある）。
逆に「他版にあって defib に無い関数」は**存在しない**。

## ただし drop-in 置換はできない

関数名は包含関係にあるが、**シグネチャが異なる**。確認済みの相違:

- `etco2` / `monitor-ecg`: `_set_title(slide, text, cap=54, maxw=20.5, breadcrumb=False)`
- `defib` / `aline`: `_set_title(slide, text, breadcrumb=False)`
- `aline`: `references(prs, refs, title="参考文献")` — 他版は `title` 引数を持たない

したがって `lib/decklib.py` で各デックの `scripts/decklib.py` を上書きしてはならない。
**現状は各デックが自分の版を同梱したまま動く構成を維持している**（そのままビルドが通ることを実証済み）。

## 収束させる場合の手順（未実施）

1. `defib` 版を基準に、`cap` / `maxw` / `title` 引数を省略可能なキーワードとして復元する
2. デックごとに 1 つずつ差し替え、`slide_lint.py` が 0 違反のままか、
   さらに**既存 pptx とタイトル文字列が全枚一致するか**を確認する
   （本移行では etco2 で 60/60 一致を確認する手順を用いた）
3. 全デックが通ってから `scripts/decklib.py` を削除し `lib/` を参照させる

この収束作業は**本移行の範囲外**である。
