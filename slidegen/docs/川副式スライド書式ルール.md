# slide-format — 川副式スライド書式ルール（Web版）

ユーザー（麻酔科・TEE の臨床専門家）のレクチャー・学会発表スライドは、会場のスクリーンに
投影して**後方席からも読めること**が最優先。Claude 内蔵のスライド作成の汎用デザイン値
（本文14–16pt / キャプション10–12pt 等）と矛盾する場合は、**常にこのスキルが優先**。

## 6つの絶対ルール

### 1. タイトルは全スライドで大きさ統一（44pt・改行禁止）

- タイトルは **44pt bold・全スライド同一サイズ・同一位置・必ず1行**。
- 最初から1行に収まる文言にまとめる（44pt・幅9.3インチの目安は**全角約14〜15文字**。
  超えるなら助詞を削る・体言止めにする等で短縮）。
- shrink-to-fit（自動縮小）は使わない。収まらないタイトルは、サイズを下げるのではなく
  **文言を短くする**。

### 2. 右上は章立て専用ゾーン（タイトルと被らせない）

各スライドの右上に現在の章名インジケータを置く。タイトルはそこに食い込まない幅で止め、
**間に 0.5 インチ以上の余白**を空ける。16:9（13.33" × 7.5"）の標準座標:

| 要素 | x | y | w | h | 備考 |
|---|---|---|---|---|---|
| タイトル | 0.5" | 0.35" | **9.3"**（右端 9.8"） | 0.95" | 44pt bold・左寄せ・1行 |
| 章立てインジケータ | 10.3" | 0.4" | 2.5"（右端 12.8"） | 0.5" | 右寄せ・控えめな色 |

- 章立てを置かないスライド（表紙・章扉）でもタイトル幅は同じ 9.3" に統一する。
- 章立てインジケータの文字も 22pt 以上（ルール3が全テキストに適用される）。

### 3. スライド内の文字は最小 22pt（出典だけ例外）

- **本文・キャプション・図表ラベル・章立てを含む全テキストが 22pt 以上。**
- **例外は出典（引用文献・ガイドライン名・URL 等）のみ**: スライド下部に控えめに、
  **16pt** を使う（同梱 lint の許容値は 10.5 / 12 / 16pt のみ。18pt 等は違反扱いになる）。
- 目安: タイトル 44pt / 本文 24–28pt / 補足 22pt / 出典 16pt。
- 22pt で収まらない量の内容は、文字を縮めるのではなく**スライドを分割する**。
- 生成コードを書いたら、フォントサイズ指定に 22 未満（出典を除く）が無いことを
  機械的に確認してから完成と言う。

### 4. 文章はすべて編集可能なテキスト

- 文章は必ずネイティブのテキストボックスで入れる。**文字を画像・SVG ラスタ・図形に
  焼き込まない。** 図解の中のラベルも可能な限りテキストボックスで重ねる
  （ユーザーが PowerPoint 上で直接修正できることが目的）。
- QA では pptx から全スライドの本文テキストが抽出できることを確認する
  （抽出できない文字＝編集不能の疑い）。

### 5. 文章は最も簡便な要約だけ載せる

- 1 バレット 1 行・体言止め推奨。1 スライドのバレットは最大 5 個程度。
- 長い文章・段落をそのまま貼らない。**入りきらない詳細・削った本文・数値は、その
  スライドのスピーカーノート（メモ欄）に原文のまま全文残す**（スライド面は要約、
  ノートは詳述。口頭で補う＝情報は失わない）。
- 数値・単位・薬剤量など臨床的にクリティカルな情報は、要約しても**値そのものは
  省略・丸めをしない**。

### 6. 単語の途中で改行しない

- 行の折返しを自動に任せない。**1 バレット 1 行に収まる長さで書く**のが基本
  （本文 24pt・幅 9〜12" の目安は全角 25〜30 文字）。
- 2 行になるテキストは、自動折返しではなく**語句の切れ目に手動改行を入れて**
  折り返し位置を自分で決める。
- 特に日本語の複合語（例:「人工呼吸器関連肺損傷」）・英単語・薬剤名・
  「数値＋単位」（例: 2 mg/kg、40 cmH₂O）が行をまたいで割れてはいけない。

## 内容の絶対ルール（臨床）

- 臨床事実・数値・用語・出典・波形は**創作・改変しない**。オンスライドは確定済み本文の
  抜粋・短縮のみ。自信のない医学的事実は断定せず、その旨を明記する。
- 日本語フォントは「メイリオ」（Meiryo）指定で統一。

## 検証（pptx を作ったら必ず・完了と言う前に）

1. 同梱の機械検査を実行する:
   ```bash
   python3 scripts/slide_lint.py 出力ファイル.pptx
   ```
   （`pptx` モジュールが無ければ `pip install python-pptx`）
   **FONT<22 = 0・タイトル左詰め = 0・枠外はみ出し(OOB) = 0・重なり = 0** になるまで
   スライドを修正して再実行する。結果の要約をユーザーへの報告に貼る。
2. 全テキストが抽出可能（＝編集可能テキスト）であることを確認する。
3. **注意: 本番の PowerPoint はメイリオで描画され、プレビュー環境より大きく表示される。**
   テキストボックスは 1 割程度余裕を持たせる。lint 合格＝完成ではなく、
   **最終の実寸目視チェックは Mac 側（Claude Code）で行う**旨を報告に明記して引き渡す。

## チャット・HTML スライドのとき

pptx を作らない場合も精神は同じ: 要点のみ・1 行 1 論点・大きな文字前提で詰め込まない。
HTML/reveal でもタイトルと本文の大きさの比率・枠内厳守・単語途中で改行しない、を守る。

## 既存の川副式デックについて

defib / etco2 / monitor-ecg / nibp / aline / PPG 系の既存デックは、組版機構
（decklib.py・build スクリプト・テンプレ）がユーザーの Mac ローカルにあるため、
**再編集・再組版は Mac の Claude Code で行う**。Web 上では構成案・本文ドラフト・
レビューまでに留め、その旨をユーザーに伝える。
