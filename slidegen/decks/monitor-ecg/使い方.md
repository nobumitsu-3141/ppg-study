# モニター心電図 スライド（作り方・直し方）

麻酔科専門医向け勉強会スライド。**川副式**デザイン（金主役・図が主役・右上パンくず・左下に出典
ハイパーリンク・ページ番号）を踏襲。8章 66枚・16:9。医学中身は Opus 執筆＋Fable 臨床チェック済み。

## 成果物
- `モニター心電図.pptx` … 完成デック（66枚）。PowerPoint/Keynote で開く（フォントは Meiryo 指定）。
- `render/モニター心電図.pdf` … 表示確認用（LibreOffice 変換・代替フォント）。

## 構成（8章）
1 歴史 / 2 成り立ち（活動電位→体表）/ 3 基礎 / 4 正常 / 5 麻酔中の見どころ /
6 異常所見 / 7 対応 / 8 特殊な使用例。各章 章扉＋内容、末尾に参考文献2枚。

## フォルダ
- `content/*.json` … 各スライドの本文・図の内容仕様・出典（Opus 執筆）。**文言を直すならここ。**
- `scripts/`
  - `figbase.py` … 配色・フォント（ヒラギノ）・図形ヘルパ。
  - `ecglib.py` … 合成心電図エンジン（方眼＋P-QRS-T＋各種調律）。**著作権セーフな模式波形**。
  - `make_figs.py` … 全図(41点)を生成 → `figs/`。図IDは `REG` を参照。
  - `decklib.py` … 川副式ビルダー（8章パンくず用に調整済み）。
  - `build_deck.py` … content＋figs → pptx。図はコードで対応（`FIGMAP`）。
- `figs/*.png` … 生成済みの図。

## 作り直し方
```
cd scripts
../.venv/bin/python make_figs.py           # 図を全部作り直す（個別は make_figs.py <id>）
../.venv/bin/python build_deck.py          # pptx を作り直す
# 表示確認:
/opt/homebrew/bin/soffice --headless --convert-to pdf --outdir ../render ../モニター心電図.pptx
```
Python は `.venv`（python-pptx / matplotlib / numpy / pillow 導入済み）。

## 直すときの勘所
- **文言・数値・出典** → `content/<章>.json` を編集して `build_deck.py` 再実行。
- **図の中身** → `make_figs.py` の該当関数（`@fig("fig_...")`）を編集して再生成。
- **波形の形** → `ecglib.py` の `beat()` パラメータ（レート・PR・QRS形・ST・T 等）。
- **章名・パンくず** → `build_deck.py` の `FULL/SHORT/SUBS/MENUSUB`、`decklib.py` の `CHAPTERS`。
- 川副式の作法（タイトル1行・重ねない・語の途中で改行しない）は iCloud の
  「PPG/スライド作成ルール.md」に準拠。

## 医学上の注意（監督メモ）
- 用量は JRC蘇生ガイドライン2020 / AHA ACLS 2020 / ASRA 2020 準拠。
  **アトロピンは JRC=0.5mg を主・AHA=1mg を併記**（勉強会の準拠先で主従を入替可）。
- 7.7（高K/TdP/LAST）・8.5（小児正常HR＝目安）は数値をそのまま断定せず「目安・準拠先に合わせる」表記。
- 波形はすべて合成（教育用の模式図）。過去問・実記録の複製は無し。
