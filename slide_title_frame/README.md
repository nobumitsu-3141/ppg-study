# SpO₂（PPG）デック タイトル枠＋日本語フォント比較

`SpO2PPG_4.16.pptx`（全47枚）を素材に、**川副式スライド書式ルールの変更案**を試作し、
あわせて**日本語フォントの比較用バージョン**を作成したもの。

## 1. ルール変更案：タイトルに「字の枠」をつける

全スライドのタイトルの背後に、**角丸の枠（フレーム）**を追加した。

| 項目 | 値 |
|---|---|
| 形状 | 角丸長方形（roundRect・角丸 約8%） |
| 線（枠） | スレート色 `#44546A`（テーマ dk2）・太さ 2.25pt |
| 塗り | なし（透明）＝枠線のみの「枠」 |
| 縦位置 | タイトル文字の行にフィット（本文帯 y≈0.66〜1.54 in） |
| 横位置 | タイトル左端の少し外側から、文字量に合わせて右へ伸縮 |

**章立てインジケータ（右上のチップ、x=8.26 in〜）と絶対に被らせない**ため、
枠の右端は最大でも **8.12 in** でクランプしている（既存の川副式ルール2「右上は章立て専用ゾーン」を踏襲）。

- タイトル文字・本文・図・出典など**既存の内容とレイアウトは一切変更していない**（枠を1つ追加しただけ）。
- タイトル文字が既に持っている袋文字（1文字ごとの黒アウトライン）はそのまま。今回の「枠」はその外側を囲む箱。

## 2. フォント比較用の3バージョン

日本語テキスト（Meiryo 指定・全3362箇所）を、各バージョンでまるごと置換した。
Latin（Arial / Calibri）は据え置き。**枠の形状・位置は3バージョンとも完全に同一**なので、純粋にフォントだけを見比べられる。

| ファイル | 日本語フォント | 位置づけ |
|---|---|---|
| `SpO2PPG_titleframe_Meiryo.pptx` | Meiryo | 基準（従来フォント＋新しい枠） |
| `SpO2PPG_titleframe_NotoSansJP.pptx` | Noto Sans JP | 比較候補① |
| `SpO2PPG_titleframe_BIZ-UDPGothic.pptx` | BIZ UDPGothic | 比較候補② |

### 比較のしかた（重要）
- フォントの見た目は**PowerPoint（Mac）で開いて**見比べること。
- 対象フォントが Mac に入っていないと代替フォントで表示され、比較にならない。事前にインストールを:
  - **Noto Sans JP**: Google Fonts から（ファミリー名「Noto Sans JP」）
  - **BIZ UDPGothic**: Windows 同梱。Mac では Microsoft/モリサワ提供分を別途インストール（ファミリー名「BIZ UDPGothic」）

## 3. 品質チェック結果（Web側）

- `validate.py`: 3ファイルとも **All validations PASSED**。
- 枠のジオメトリ: 章立てチップとの重なり **0**、スライド外はみ出し **0**（枠右端 ≤ 8.12 in ＜ チップ 8.26 in）。
- `slide_lint.py`: **枠追加による新規違反 0 件**（元デッキと違反明細が完全一致）。
  既存の違反（FONT<22・本文あふれ・重なり 等）は**元の Mac 作成デッキ由来**で、Web 側の推定値による近似のため今回は手を触れていない（内容・レイアウトの改変になるため）。
- 本文テキスト: 元デッキと**完全一致**（枠は空のシェイプなので文字は増減なし）。

## 4. 注意事項

- **スライド22** のみ、元デッキで図がタイトル帯（y=1.30 in）まで食い込む構成のため、枠の下辺が図の上端付近を横切る。元デッキでもタイトルが図に重なる箇所であり、内容は不変。気になる場合は Mac 側で図/枠を微調整のこと。
- 本番の PowerPoint はメイリオ等で描画され、プレビューより大きく表示される。**枠と文字の実寸最終確認は Mac の PowerPoint で**行うこと（Web 環境では LibreOffice レンダリングが不可・対象フォント未導入のため画像確認はしていない）。
- このルール変更を恒久化する場合は、Mac 側の `slide-format` スキル（SKILL.md）の「絶対ルール」にタイトル枠の項を追記する。

## 5. 再生成

`build_titleframe.py` を、元 `SpO2PPG_4.16.pptx` を `unpacked/` に展開した状態で実行すると3ファイルを再生成できる。
枠の色・太さ・角丸・帯の位置は同スクリプト冒頭の定数で調整可能。
