## 目視QA（全47枚レンダリング）の結論
**視覚的には全スライド・クリーン**（文字切れ・実際の衝突・崩れなし）。タイトルは全ページ 44pt。
残り 121 件の lint は**大半が偽陽性/保守的判定**で、見た目の問題ではない:

- **本文あふれ ~7 件は lint バグ**: 章扉アジェンダ(p2/5/10/25/30/33/36)は 38pt の固定行間
  (spcPts)を使い、`est_overflow` がそれを行間倍率と誤解して "+43,000km" と誤計算する。
  実際は枠内に余裕で収まる（レンダリングで確認）。
  → 本筋は `lib/slide_lint.py` の `est_overflow` を「`line_spacing` が Emu(=固定pt)なら
  絶対値扱い」に直すこと（他デックの 0 違反には影響しない。別対応可）。
- **重なり ~50 件の大半**: 波形図など**bbox の広い図の“余白”に注釈ラベルがある**ための
  判定で、レンダリング上は文字と曲線が衝突していない（例 p18・p19）。移動はむしろ改悪。
- **FONT<22=41 / 折返し=14**: 図に囲まれた密な枠。可読だが 22pt には分割/図再配置が要る
  （p24 6分類図・p33 3×3マトリクスは分割前提）。現状でも判読可能。
- **パンくず侵入=1（p19「3.5 大血管エラスタンス」）**: 臨床用語で短縮不可・見た目は非衝突。

## 環境メモ（当セッションで解消済み）
当初 LibreOffice が「source file could not be loaded」で全変換に失敗＝目視QA不可だった。
原因は**部品欠品**（`libreoffice-impress`/`-writer` 未導入で、pptx/txt のインポートフィルタが無い）。
`apt-get install libreoffice-impress` で解消し、全47枚をレンダリング・目視できた。
※ Web の描画は Noto 代替で、本番メイリオより字面がやや狭い。**最終の実寸目視は Mac** が確実。
