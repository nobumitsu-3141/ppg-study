## 残り（229件）＝ Mac で仕上げるべき理由
- **図ラベルの 22pt 化（FONT<22=119 の主因）**: 枠拡大・文言短縮・スライド分割が要る
  （p24 6分類図・p33 3×3マトリクスは分割前提）。
- **あふれ33・折返し27・重なり49**: レイアウト再配置の判断が要る。重なりの多くは
  **図の上に載せた注釈ラベル**で、消すには図外へ動かす＝設計判断。
- **パンくず侵入1（p19「3.5 大血管エラスタンス」）**: 臨床用語のため短縮不可。
  本筋はパンくず帯（右上チップ列）の再設計（全ページ・要デザイン判断）。
- **この環境では LibreOffice が壊れており目視レンダリング QA ができない**。上記の
  レイアウト変更は目視確認が必須のため、**Mac(メイリオ＋render)で仕上げる**のが確実。

## Mac での続き方
1. `SpO2PPG_revised_websafe.pptx` を起点にする（安全な下地は適用済み）。
2. `SpO2PPG_slide_revision_spec.md` のスライド別指示に従い、残りを目視しながら修正。
3. `python3 slidegen/lib/slide_lint.py <pptx>` が 0 になるまで反復、最後にメイリオで実寸目視。
