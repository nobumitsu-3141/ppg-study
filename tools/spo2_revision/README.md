## 残り 120 件の内訳と、なぜ Web で自動処理しないか
- **重なり=49**: ほぼ全部が **図の上に載せた注釈ラベル / 重なる図同士**（p22・p24 は図×図）。
  消すにはラベルを図の外へ動かす＝**どの特徴を指すか**の設計判断。目視なしでは劣化リスク大。
- **本文あふれ=15 のうち約7件は lint の偽陽性**: 章扉アジェンダ(p2/5/10/25/30/33/36)は
  **38pt の固定行間(spcPts)** を使用。`slide_lint.py` はこれを「行間倍率」と誤解し行高を
  天文学的に計算する（"+4,300,285cm" 等）。実際は 9 行×38pt=約12cm で 15cm 枠に収まる。
  → **デック側は正常**。slidegen 側 `lib/slide_lint.py` の `est_overflow` を
  「`line_spacing` が Emu(=固定pt)なら絶対値として扱う」よう直すのが本筋（別対応可）。
- **FONT<22=41 / 実あふれ約8 / 折返し=14**: 図に囲まれ**空きの無い枠**。22pt 化・収めるには
  スライド分割 or 図の再配置が必要（p24 6分類図・p33 3×3マトリクスは分割前提）。
- **パンくず侵入=1（p19「3.5 大血管エラスタンス」）**: 臨床用語で短縮不可。
  本筋はパンくず帯（右上チップ列）の再設計（全ページ・要デザイン判断）。
- **この環境は LibreOffice が壊れ目視レンダリング QA 不可**。上記は目視確認が必須のため、
  **Mac(メイリオ＋render)で仕上げる**のが確実。

## Mac での続き方
1. `SpO2PPG_revised_websafe.pptx` を起点にする（安全な下地は適用済み）。
2. `SpO2PPG_slide_revision_spec.md` のスライド別指示に従い、残りを目視しながら修正。
3. `slidegen/lib/slide_lint.py` を実行（アジェンダの偽陽性は上記のとおり）→ メイリオで実寸目視。
