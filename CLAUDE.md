# ppg-study — 作業指示

麻酔科専門医（離島の一人麻酔科）が著者。**PPG（photoplethysmography：光電容積脈波）**
の波形形態から非侵襲的に血管トーンを推定できるかを文献にもとづき検証した文書群と、
講義スライドの組版機構からなる。

このリポジトリには性格の異なる 2 つの領域がある。**触る前にどちらかを見極めること。**

| 領域 | 場所 | 中身 |
|---|---|---|
| 研究ドキュメント | ルート直下 `*.html` / `*.md` | GitHub Pages で公開。`index.html` がハブ |
| スライド組版機構 | `slidegen/` | 川副式 pptx のビルダー。詳細は `slidegen/README.md` |

---

## 1. 全領域に共通する絶対ルール

### 1.1 臨床内容を創作・改変しない

- **臨床事実・数値・単位・薬剤量・波形・出典を、確認なしに書き換えない。**
- PMID・DOI はすべて個別に検証済みである。**タイポに見えても直さない。**
- 過去資料の誤りを意図的に訂正した箇所が複数ある。古い資料と食い違って見えても
  **本リポジトリの記載が正しい**。元に戻さないこと。
- 「未確認」「留保」と明記された箇所は**意図的な正直さ**である。断定形に書き換えない。
- 内容（事実・数値・引用・論旨）に疑問が出たら、**自分で判断せず著者に確認する。**
  形式（HTML 実装・CSS・レイアウト）は既存の慣例に合わせて判断してよい。

### 1.2 文体

- **である調**（です・ます調は使わない）
- タイトルは「本題 ― 副題」形式（全角スペース＋ em dash ＋全角スペース）
- **専門用語の初出は英語フルスペル＋日本語訳を必ず併記**（著者の明示的な要求）
- 高度な数学・物理は「まず高校レベル → 次に正確な説明」の 2 段階で書く
- 臨床・研究の記述には**必ず URL 付きの文献的裏づけ**を添える

## 2. スライドを作業するとき（`slidegen/`）

### 2.1 最初に読むもの

`slidegen/docs/川副式スライド書式ルール.md`。**Claude 内蔵の汎用スライド設計値
（本文 14–16 pt 等）と矛盾する場合、常にこのルールが優先する。**
会場のスクリーンに投影して後方席から読めることが最優先だからである。

要点:

- タイトルは **44 pt 固定**・全スライド同一・**1 行**。収まらなければ**文言を短くする**（縮小しない）
- 全テキストが **22 pt 以上**。例外は出典のみ（10.5 / 12 / 16 pt）
- **1 行 1 論点**。1 スライドのバレットは最大 5 個程度
- **単語の途中で改行しない**。複合語・薬剤名・「数値＋単位」が行をまたいで割れてはいけない
- 文章は必ず編集可能なテキストボックスで入れる。**画像に焼き込まない**
- 入りきらない詳細・数値は**スピーカーノートに原文のまま全文残す**

### 2.2 初期化（Linux / claude.ai では必須）

```bash
bash slidegen/setup-linux.sh
```

Python 依存に加え、**Meiryo → Noto Sans CJK JP の対応づけ**を入れる。
これを省くと漢字が**中国語字体（簡体字 SC）**で描画される。

### 2.3 build と検証

```bash
cd slidegen/decks/etco2/scripts
python3 make_figs.py && python3 build_etco2_deck.py     # → ../out/*.pptx
python3 ../../../lib/slide_lint.py ../out/*.pptx        # 0 違反になるまで直す
python3 ../../../tools/render_check.py ../out/*.pptx --slides 1,5,12   # 目視確認
```

**lint が 0 でも「完成」ではない。** Linux の描画は代替フォントによるもので、
本番の PowerPoint（メイリオ）より字面が小さい。ここで「ちょうど収まっている」ものは
本番で溢れうる。**最終の実寸確認は Mac の PowerPoint で行う**旨を必ず報告に明記して引き渡すこと。

### 2.4 やってはいけないこと

- `lib/decklib.py` で各デックの `scripts/decklib.py` を上書きしない
  （4 分岐しておりシグネチャが異なる。`slidegen/docs/decklib-forks.md` 参照）
- `figs/` の図を Linux で再生成して**そのままコミットしない**。
  macOS のヒラギノで作られた既存の図と字形が変わり、1 デック内で字形が混在する
- `decks/nibp/build_deck.py` の `_TITLE_METRIC_FONT` を、断りなく書き換えない

## 3. 研究ドキュメントを作業するとき（ルート直下）

- 雛形は `PPG_dicrotic_notch_resistance_review.html` と `PPG_LVET_six_factors_figures.html`。
  **`t2_pi_normalization.html` を雛形にしない**（配色が旧世代の外れ値）
- 配色は `--paper:#F4F7F7` / `--accent:#0B7A6E`（ティール）/ `--warn:#A8602E`（オークル）
- フォントは Space Grotesk（見出し）/ Newsreader（本文）/ IBM Plex Mono（ラベル）を
  **base64 で自己完結埋め込み**。外部 CDN 参照なし・オフライン動作が設計目標
- **MathJax・KaTeX は使用禁止。** 数式は CSS ベースで組む
- 外部ライブラリは使わない（vanilla JS のみ）
- **仮説文書は視覚的に区別する。** オークル系コールアウト＋「仮説提示」タグ。
  臨床上の誤読を避けるための要件であり装飾ではない
- 各文書の末尾は「実務メモ（一人麻酔科の視点）」＋免責文。省略しない

`index.html` だけは Google Fonts の CDN を使う既知の非対称がある。**そのまま維持する。**
